## GitHub Release Title

`buddy-voice v1.0.0`

## GitHub Release Description

`buddy-voice` gives Claude Code's `/buddy` companion a real voice.

This release includes:

- a polished README built around the product's magic moment
- zero-install setup via `CLAUDE.md`
- a standalone CLI for direct buddy conversations
- a VS Code extension that detects buddy speech locally and speaks it aloud
- contributor docs and repo metadata cleanup
- ready-to-use GitHub social preview assets

Why it matters:

Your buddy already has personality. This makes it audible with local macOS text-to-speech, without rewriting Claude Code or sending audio off your machine.

Highlights:

- local-first audio with macOS `say`
- hands-free terminal narration in VS Code
- lightweight CLI chat mode
- clearer install paths
- share-ready project branding and release assets

## VS Code Marketplace Blurb

Give your Claude Code buddy a voice in VS Code.

Frostwig Voice watches your terminal locally, detects buddy speech, and speaks it aloud with macOS text-to-speech. No extra cloud audio service, no terminal injection, and no telemetry. Just the `/buddy` experience, but louder.

Best for:

- developers running Claude Code in VS Code
- buddy fans who do not want to miss quips while coding
- anyone who wants a local-first TTS layer instead of a complicated audio stack

Key features:

- local buddy speech detection
- macOS native TTS
- status bar mute toggle
- voice picker and test command
- buddy stats command
